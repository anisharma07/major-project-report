\section{Performance Metrics}

The trained models were evaluated based on multiple metrics to assess their suitability for real-time parking detection across different deployment scenarios.

\subsection{Evaluation Criteria}

\begin{itemize}
    \item \textbf{Precision:} Ratio of true positive detections to all positive detections (accuracy of detected vehicles)
    \item \textbf{Recall:} Ratio of true positive detections to all actual vehicles (completeness of detection)
    \item \textbf{F1-Score:} Harmonic mean of precision and recall (balanced performance metric)
    \item \textbf{Inference Time:} Processing time per frame in milliseconds
    \item \textbf{mAP@0.5:} Mean Average Precision at IoU threshold 0.5
\end{itemize}

\subsection{Analysis of Results}

\subsubsection{YOLOv8n Edge Performance}
The YOLOv8n model demonstrated exceptional performance on the Raspberry Pi:

\begin{itemize}
    \item \textbf{Precision:} Achieved 0.99 precision, indicating minimal false positivesâ€”critical for user trust
    \item \textbf{Inference Speed:} 7.7ms inference time enables real-time processing at 130 FPS theoretical maximum
    \item \textbf{F1-Score:} 0.933 shows excellent balance between precision and recall
    \item \textbf{Trade-off:} Slightly lower recall (0.882) acceptable for edge deployment given the speed advantage
\end{itemize}

The high precision with minimal inference time makes YOLOv8n ideal for edge deployment, where computational resources are limited but accuracy remains paramount for practical deployment.

\subsubsection{YOLOv8m and YOLOv8l Performance}
Larger models trained on extensive datasets showed strong overall performance:

\begin{itemize}
    \item \textbf{YOLOv8m:} Good balance with F1-score of 0.927, but 257ms inference time limits real-time applications on edge devices
    \item \textbf{YOLOv8l:} Highest recall (0.950) and F1-score (0.942), suitable for cloud-based processing where computational resources are abundant
    \item \textbf{Use Case:} Better suited for offline analysis, batch processing, or cloud-hosted inference
\end{itemize}

\section{Limitations and Observations}

\subsection{Identified Challenges}

\begin{itemize}
    \item \textbf{Occlusions:} Large vehicles (buses, trucks) can partially obscure adjacent spots
    \item \textbf{Irregular Layouts:} Non-standard parking arrangements require custom grid configuration
    \item \textbf{Lighting Variations:} Significant performance degradation in very low light without artificial illumination
    \item \textbf{Vehicle Type Diversity:} Motorcycles and bicycles sometimes missed due to size
\end{itemize}

\subsection{Mitigation Strategies}

\begin{itemize}
    \item \textbf{Camera Placement:} Elevated mounting reduces occlusion issues
    \item \textbf{Adaptive Thresholds:} Configurable confidence thresholds per deployment
    \item \textbf{Temporal Smoothing:} Multiple frame averaging to reduce false detections
    \item \textbf{Night Vision:} Infrared cameras for 24/7 operation consideration
\end{itemize}
